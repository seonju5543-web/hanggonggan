# 한장 — 수익화 전략 (Profitability Plan)

> 원칙: **학생(주 고객)에게는 영원히 무료.**
> 토스가 송금 무료로 사용자를 모은 뒤 금융 중개로 수익화했듯,
> 한장은 장학금 매칭·신청 무료로 사용자를 모은 뒤 **양면 시장의 반대편**(대학·재단·기업)에서 수익을 만든다.

## 1. 시장 구조

| 플레이어 | Pain Point | 한장이 파는 것 |
|---|---|---|
| 학생 (무료) | 장학금이 흩어져 있고 절차가 복잡 | 매칭 + 원클릭 신청 (무료 = 성장 엔진) |
| 대학 장학팀 | 공고 관리·서류 접수·심사가 수작업 | B2B SaaS |
| 장학재단·기업 CSR | 적격 지원자 모집이 어려움 | 타겟 도달·접수 인프라 |
| 금융사 | 대학생 고객 획득 비용(CAC)이 높음 | 제휴 채널 |

## 2. 수익원 (우선순위 순)

### ① 재단·기업 공고 게재 및 타겟 매칭 수수료 — *가장 빠른 수익*
- 장학재단은 "조건에 맞는 지원자"를 모집하는 데 어려움을 겪음 (지원 미달·부적격 지원 다수)
- 한장은 조건이 검증된 학생에게만 공고를 노출 → **적격 지원자당 과금(CPA)** 또는 공고 게재비
- 광고가 아니라 매칭이므로 학생 경험을 해치지 않음 (요건 미충족 학생에게는 노출 자체가 없음)
- 예상 단가: 공고 건당 게재비 50~300만원 / 적격 지원자당 5,000~20,000원

### ② 대학 장학팀 B2B SaaS — *가장 큰 매출*
- 공고 등록 → 지원자 접수 → 서류 검증 → 심사 → 선정 통보를 페이퍼리스로
- 학교 입장에서는 행정 인건비 절감 + 장학금 집행률(대학 평가지표) 개선
- 과금: 재학생 수 기준 연간 구독 (예: 재학생 1만 명 대학 연 1,200~3,000만원)
- 국내 4년제 대학 190여 개 × 평균 2,000만원 = TAM 약 38억원/년 (전문대 포함 시 2배)

### ③ 금융·생활 제휴 (토스 모델) — *스케일 수익*
- 학자금 대출 비교·중개 수수료 (국가 학자금대출 외 보완 대출)
- 대학생 특화 적금·체크카드·통신 요금제 제휴 (가입당 수수료 1~3만원)
- 어학시험·인강·자격증 제휴 할인 (판매 수수료)
- **신뢰 원칙**: 장학금 매칭 결과와 광고를 절대 섞지 않는다. 제휴 상품은 별도 탭에서만.

### ④ 익명 통계 데이터 리포트 — *부가 수익*
- "전공별·소득구간별 장학금 수혜 격차" 등 익명·집계 데이터 리포트를 대학·정책기관·연구소에 판매
- 개인정보보호법 준수: 개인 단위 데이터 판매 금지, 집계·비식별 데이터만

### ⑤ 하지 않을 것
- 학생 대상 유료 구독/프리미엄 매칭 (핵심 가치를 유료화하면 성장이 죽음)
- 배너 광고 (신뢰 자산 훼손)
- 개인 데이터 판매

## 3. 실행 로드맵

| 단계 | 목표 | 수익 활동 |
|---|---|---|
| Phase 1 (0~6개월) | 1~3개 대학에서 MAU 5천 | 없음 — 사용률·매칭 정확도에 집중 |
| Phase 2 (6~12개월) | 10개 대학, MAU 3만 | ① 재단 공고 게재 파일럿 (재단 5곳) |
| Phase 3 (12~24개월) | 50개 대학, MAU 15만 | ② 대학 SaaS 유료 전환, ③ 금융 제휴 개시 |
| Phase 4 (24개월~) | 전국 커버리지 | ④ 데이터 리포트, 대학원·유학 장학 확장 |

## 4. 핵심 지표 (North Star)
- **North Star: 사용자당 연간 수혜 확정 장학금액** (이 숫자가 크면 모든 수익원이 따라옴)
- 보조: 온보딩 완료율, 매칭→신청 전환율, 신청→선정률, 재단 공고 재게재율

## 5. 규제 체크리스트
- 개인정보보호법: 소득구간·성적은 민감정보 수준으로 취급, 암호화 저장 + 목적 외 이용 금지
- 금융 제휴 시: 금융상품 판매대리·중개업 등록 또는 등록 제휴사 경유
- 대학 데이터 연동: 각 대학 개인정보 제3자 제공 동의 프로세스 필요
