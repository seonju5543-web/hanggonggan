요약 (Executive Summary)

「한대장」은 흩어져 있는 교내·교외 장학금을 학생의 조건으로 자동 매칭하고, 증명서류 준비와 공고 원본 신청서(HWP)의 디지털 작성까지 앱에서 끝내는 모바일 플랫폼입니다. 토스가 송금의 마찰을 없앴듯, 한대장은 장학금 신청의 마찰을 없앱니다.

아이디어 단계가 아닙니다 — 제품은 배포되어 작동 중입니다. 전국 23개 캠퍼스 수집망이 매일 실제 공고를 모으고, 실제 공고 18건이 자격 진단·양식 작성까지 지원되며, 실공고 신청서 7종이 원본과 동일한 구조로 디지털화되어 있습니다.

요청: G-RISE 프로그램 선정과 지원금 200만 원(사용 계획 Ⅴ절 — 한도 내 조정 가능), 그리고 사업화·개인정보 법률 멘토링 및 교내(학생지원팀) 미팅 주선입니다.

Ⅰ. 문제 정의 — 비어 있는 연결 계층

표 1. 장학금 시장의 구조적 마찰
마찰내용
정보 분산공고가 학교 본부·단과대·학과·외부 재단·지자체로 흩어져 전수 파악이 사실상 불가능
기회 손실자격이 되는 학생이 공고를 몰라서, 또는 마감을 놓쳐서 신청하지 못하는 구조적 누락
반복 마찰장학금마다 자격 확인 → 증명서 발급 → 상이한 HWP 양식 작성 → 제출을 처음부터 반복
행정 비효율장학팀은 부적격 신청·서류 미비·반복 문의에 행정력 소모

재원은 존재하고 수요도 존재하지만, 둘을 잇는 매칭·준비 계층(matching & preparation layer)이 비어 있습니다. 한대장은 이 계층을 차지하는 서비스입니다.

Ⅱ. 솔루션 — 제품 상세

표 2. 핵심 기능 (전부 현재 가동 중)
기능내용
1분 온보딩학교(전국 55개 자동추천)·캠퍼스(이원화 13개교)·계열·학년·평점·소득구간·특별자격 1회 등록
맞춤 매칭 엔진12개 신호 자격 진단 → 4단계 판정 + 요건별 사유 표시 + 적합도 점수(0~99%) 순위화
공고 자동 수집망일반 수집기 + 실제 브라우저 수집기(봇 차단 게시판용)가 매일 각 대학 게시판을 확인해 자동 게재
원본 양식 디지털 작성공고 첨부 신청서(HWP)를 스키마화 → 문답형 작성(프로필 자동 채움) → 원본과 동일 구조의 문서 생성·인쇄·공유
서류 보관함증명서류를 기기 내 저장(서버 미보관) — 신청마다 자동 체크리스트로 재사용
진행 단계 관리신청 준비 → 공식 제출 → 심사 → 선정 발표 4단계를 학생이 앱에서 관리
자동 업데이트새 장학금·양식이 서버 데이터만으로 설치된 앱에 즉시 반영 (재설치 불요)

※ 차별점: 단순 "공고 모음"이 아니라, 실제 신청서 양식까지 디지털로 재현하는 것은 국내에서 한대장이 유일합니다. 모든 화면은 서비스 주소에서 직접 확인 가능합니다.

Ⅲ. 현재까지의 성과 (Traction)

표 3. 성과 지표 (2026. 7. 기준)
항목성과
제품 배포설치형 모바일 웹앱(PWA) 정식 배포 — 매일 자동 갱신 운영 중
수집망전국 23개 캠퍼스 등록, 일반+브라우저 이중 수집기 매일 2회 가동 (한국외대 게시판 포함)
실공고 데이터실제 공고 18건 정식 등록(자격 진단·마감 D-day·원본 첨부), 실시간 피드 44건+
양식 디지털화실제 공고 신청서 7종을 원본 동일 구조로 재현 (한국장학재단·성균관대·명지대 공고 등)
보안·개인정보개인정보 전량 기기 내 저장(서버 미보관) 설계 + 전체 코드 보안 점검 완료
운영 자동화수집→검증→게재→만료 삭제까지 무인 파이프라인 — 운영 인건비 0
