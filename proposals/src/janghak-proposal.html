요약 — 무엇을 요청드리는가

「한대장」은 학생이 학적·성적·소득구간을 1회 등록하면 교내·교외 장학금을 자동 매칭하고, 증명서류 준비와 공고 첨부 신청서(HWP)의 디지털 작성까지 앱에서 끝내는 서비스입니다. 본교 재학생(제안자)이 개발해 이미 배포·작동 중이며, 본교 장학공지 게시판도 매일 자동 수집하고 있습니다.

요청드리는 것은 세 가지입니다 — ① 장학공지 공식 채널 확인(비용·협약 불요), ② 2026-2학기 교내 장학금 1~3종의 표준화 접수 파일럿, ③ 시연을 겸한 30분 미팅.

학교 부담 비용은 없으며(전 과정 무료), 파일럿 범위·중단 권한은 전적으로 장학팀에 있습니다.

Ⅰ. 문제 — 학생과 장학팀 모두가 겪는 비용

장학 공고는 학교 본부·단과대·학과·외부 재단·지자체로 분산되어 있어, 자격이 충분한 학생이 공고 자체를 인지하지 못해 신청 기회를 놓치는 일이 구조적으로 반복됩니다. 학생은 장학금마다 공고 탐색 → 자격 확인 → 증명서 발급 → 상이한 양식(HWP) 작성 → 제출을 처음부터 반복하고, 장학팀은 그 반대편에서 반복 비용을 치릅니다.

표 1. 양측이 반복 부담하는 비용
주체반복 비용
학생공고 미인지로 인한 신청 누락 · 마감 임박 인지 · 자격 요건 해석의 어려움 · 매번 다른 HWP 양식 작성 · 증명서 중복 발급
장학팀동일한 자격 문의 응대 반복 · 부적격 신청 반려 · 서류 미비 보완 요청 · 형식이 제각각인 신청서 검토

※ 요컨대 "재원은 준비되어 있는데 연결이 안 되는" 문제이며, 연결 계층은 시스템으로 채울 수 있습니다.

Ⅱ. 한대장 소개 — 이미 작동하는 해결책

한대장은 기획서가 아니라 배포되어 작동 중인 서비스입니다. 홈 화면에 설치하는 모바일 웹앱(PWA)으로, 아래 기능이 현재 가동되고 있습니다.

표 2. 현재 가동 중인 기능 (2026. 7. 기준)
기능내용
맞춤 매칭 엔진학적 12개 신호로 자격 진단(충족·미충족 사유 표시) + 적합도 점수 순위화
공고 자동 수집전국 23개 캠퍼스 장학 게시판을 매일 아침 자동 확인 — 본교 학생지원팀 장학공지 게시판 연결 완료
원본 양식 디지털 작성공고 첨부 신청서(HWP)를 문답형으로 작성 → 원본과 동일한 구조의 문서 생성·인쇄·공유 (현재 실제 공고 양식 7종 지원)
서류 보관함증명서류를 학생 기기에만 저장(서버 미보관) — 신청 시 자동 체크리스트
진행 단계 관리신청 준비 → 공식 제출 → 심사 → 발표까지 학생이 자신의 진행 상황을 앱에서 관리
정직 표기 원칙실존 공고만 게재 · 접수가 실제로 일어나지 않는 것을 "신청 완료"로 표기하지 않음

※ 실제 공고 18건이 자격 진단·양식 작성까지 지원되는 상태로 등록되어 있으며(성균관대·광운대·명지대·홍익대·상명대 공고 포함), 매일 자동으로 갱신됩니다.

표 3. 기대 효과
구분기대 효과
학생 지원공고 인지율 상승 → 신청·수혜 기회 확대, 마감 놓침 방지
행정 효율자격 사전 진단으로 부적격 신청·서류 미비 접수 감소, 반복 문의 응대 부담 경감

Ⅳ. 협력 방안 — 3단계 파일럿

1단계 · 정보 연계즉시 가능 · 협약 불요
장학공지 게시판을 공식 수집 채널로 확인 → 학생이 앱에서 본교 공고를 실시간 확인
→

지정 장학금 1~3종을 표준 양식+이메일로 접수 → 통일된 형식으로 검토 부담 감소
→

그림 1. 3단계 파일럿 구조 — 각 단계는 독립적이며, 다음 단계 진행 여부는 장학팀이 결정합니다

1단계는 이미 기술적으로 완료되어 있어 확인만으로 즉시 시작됩니다. 2단계는 장학팀이 지정한 소수의 장학금에서 한 학기만 시범 운영하고, 효과(신청 건수·서류 미비율·문의 감소)를 수치로 정리해 보고드립니다. 3단계는 파일럿 성과가 확인된 뒤에만 협의합니다.

Ⅴ. 요청 사항 (구체)

요청 ① — 장학공지 공식 채널 확인 (소요: 회신 1통 · 비용 없음)

학생지원팀 장학공지 게시판(dep.hufs.ac.kr 학생지원 공지)을 한대장의 공식 수집 채널로 확인해 주십시오.

이미 매일 자동 수집 중이므로, "해당 게시판이 장학 공고의 공식 게시 위치가 맞다"는 확인과, 누락되는 공고 유형(학과 게시판 등 별도 경로)이 있다면 알려주시는 것으로 충분합니다.

요청 ② — 표준화 접수 파일럿 (2026-2학기 · 1~3종 한정)

2026-2학기 교내 장학금 중 1~3종을 지정해 주시고, 해당 건에 한해 "앱이 생성한 표준 신청서(공고 원본 양식과 동일 구조) + 증명서류"의 이메일 접수를 인정해 주십시오.

필요한 것은 접수용 이메일 주소 1개(기존 장학팀 메일 사용 가능)와 대상 장학금 지정뿐입니다. 접수 형식·명명 규칙은 장학팀 기준에 맞춰 조정하며, 파일럿 중단 권한은 언제든 장학팀에 있습니다.

요청 ③ — 30분 미팅 (7월 중 · 시연 포함)

위 두 가지를 확정하기 위한 30분 미팅을 요청드립니다. 실제 앱 시연과 파일럿 운영안(대상 장학금 후보·접수 형식·성과 지표)을 준비해 가겠습니다.

Ⅵ. 개인정보 보호와 책임 범위

  • 서버 미보관: 학생의 프로필·증명서류·양식 답변은 학생 기기 안에만 저장됩니다. 외부로 전송하는 코드가 없으며, 자료가 기기를 떠나는 것은 학생이 직접 공유·제출할 때뿐입니다.
  • 보안 점검: 외부 스크립트 차단(CSP)·수집 데이터 오염 대비 방어·접속 기록 미전송 등 보안 점검을 완료했습니다.
  • 정직 표기: 실제 접수가 일어나지 않는 단계를 "신청 완료"로 표기하지 않습니다. 파일럿 이전까지 최종 제출은 항상 공식 채널로 안내합니다.
  • 책임 범위: 파일럿 기간 중 발생하는 이슈의 수정 책임은 제안자에게 있으며, 장학팀은 언제든 파일럿을 중단할 수 있습니다.

맺음말

한대장은 본교 재학생이 "우리 학교 학생들이 장학금을 놓치지 않게 하자"는 문제의식으로 만들어 이미 작동하고 있는 서비스입니다. 학교의 새로운 예산이나 시스템 구축 없이, 확인 1통과 메일 주소 1개로 학생들의 장학 접근성이 즉시 달라질 수 있습니다. 짧은 미팅 기회를 주시면 실제 화면으로 보여드리겠습니다. 감사합니다.
