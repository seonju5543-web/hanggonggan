## 서비스 개요
**한대장(한국 대학교 장학금)**: 대학생이 프로필 1회 등록으로 교내·교외 장학금을 자동 매칭받고,
증명서류 준비와 **공고 원본 양식 작성**까지 앱에서 끝내는 모바일 PWA.
- 배포 주소: https://seonju5543-web.github.io/hanggonggan/ (main에 push하면 자동 배포)
- 학생 영구 무료. 수익화 전략은 `BUSINESS.md`.

## 운영 원칙 (개발자와 합의됨 — 변경 금지)
1. **정직한 신청 상태**: 실제 접수가 일어나지 않는 것을 "신청 완료"로 표기하지 않는다.
   현재 모델 = "신청 준비 완료" + 최종 제출처 안내. 이메일 접수 공고만 실제 전송 가능.
2. **장학금 정식 등록(매칭+양식 지원)은 개발자 컨펌 후에만**. 컨펌은 **채팅에서 개발자가 직접 지시**한다
   (GitHub 이슈는 열람용 메뉴판일 뿐, 컨펌 행위는 채팅에서).
   실시간 공고 피드(제목+링크)는 컨펌 없이 자동 게재 — 이건 합의된 예외.
3. **선조치후보고**: 방향 결정이 필요하면 최적안으로 먼저 조치하고 "어떤 상황에서 어떤 조치"를 보고.
4. **양식의 정의**: 공고에 첨부된 실제 신청서 파일(HWP 등)과 **동일한 구조**의 문서를 생성해야 한다.
   자유 서식 텍스트로 대체하면 안 됨. 신규 양식은 스키마화(`forms.js`) 후 등록.
5. **양식 확보 절차 (개발자 컨펌 이후)**: ① 공고 상세의 첨부 양식 원본을 fetch-page 워크플로로 확보
   ② 원본 항목·문구·체크옵션 그대로 스키마화 ③ 검증 후 배포.
   제출 방식이 양식 제출이 아니거나(포털 입력형 등) 양식 확인에 문제가 있으면,
   **모든 수단(공고 상세, 첨부, 주관 재단 홈페이지, fetch-page 원격 열람, 정찰 도구)을 동원해 찾아본 뒤**
   그래도 못 찾을 때만 개발자에게 필요한 정보를 정리해 문의한다.

## 중요한 기술 사실 (재발견에 시간 쓰지 말 것)
- 이 저장소 **기본 브랜치는 `claude/nice-heisenberg-WESq5`** (예약 워크플로는 여기 있어야 작동).
  **Pages 배포는 main 브랜치 소스** — 배포하려면 main에 push. 작업 브랜치는 `claude/korean-scholarship-app-8posyu`.
  세 브랜치를 모두 같은 내용으로 push하는 것이 관례 (main push 전 `git fetch origin main` — 봇이 커밋함).
- Claude 실행 샌드박스는 학교 사이트·github.io 접속 차단 → 외부 확인은 GitHub Actions(로봇/정찰/fetch-page)로.
- iOS Safari는 datalist 미지원(커스텀 자동추천 구현됨), 전역 appearance:none이 체크박스 지움(커스텀 체크 구현됨).
- 검증: `python3 -m http.server 8123` + scratchpad의 Playwright 드라이버(chromium: /opt/pw-browsers/chromium-1194/chrome-linux/chrome). 한글 파일명 setInputFiles 실패함(ASCII 사용).

## 전체 현황 (2026-07-04 기준 — 3단 정리)

### 1. 완성된 것 (배포되어 작동 중)
- **앱 배포**: PWA(홈 화면 설치형), https://seonju5543-web.github.io/hanggonggan/ — main push 시 자동 배포
- **온보딩**: 학교 자동추천(별칭 "외대"·"서울" 지원) → 캠퍼스(이원화 13개교) → 계열 → 학과 자동추천(학교 반영) → 성적·소득 → 특별자격(체크 표시) → 공통 서류정보(학번·연락처·계좌)
- **매칭 엔진**: 11개 신호 자격 진단 + 4단계 판정 + 요건별 사유 표시 + 적합도 점수(0~99%) 정렬
- **실시간 공고 피드**: 수집 로봇이 매일 09:00 KST 10개 캠퍼스 게시판 수집 → 공고 상세 방문 → 첨부양식·마감단서 채집 → 앱 '장학금' 탭에 자동 표시 (실공고 40건 발행 중)
- **양식 엔진**: 실제 공고 양식과 동일 구조 문서 생성 — 질문형 입력 → 체크박스·시간표·서약문까지 재현 → 수정 → .doc 저장/인쇄/공유 (등록 1종: kosaf-ai-mentor 활동계획서)
- **서류 보관함**: 증명서류 8슬롯 기기 내 저장, 신청 시 자동 첨부 체크리스트, 재사용
- **신청 준비 플로우**: 단건/일괄, 자유서식 도우미(자소서·계획서·사유서), 제출처별 단계 가이드, 복사·파일 공유, 접수메일 자동완성(mailto)
- **컨펌 파이프라인**: 리포트 이슈 자동 생성(개발자 담당 지정) → 채팅 컨펌 → 등록
- **인프라**: 정찰 도구(probe), 원격 열람(fetch-page), 자동발송 서버 코드(server/)
- **문서**: 제안서 2종(proposals/ — 장학팀·G-RISE DOCX), BUSINESS.md, 본 인수인계 문서
